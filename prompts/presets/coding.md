Additional context for this session: the prompts are for a coding assistant working
in the speaker's codebase.
- Keep concrete technical details exactly as spoken: file names, paths, function
  names, error messages, library names, versions.
- If the speaker described a bug, structure the prompt as: what happens, what should
  happen, and any clues about where to look.
- Preserve constraints as explicit requirements (e.g. "don't change the public API",
  "keep the tests passing").
